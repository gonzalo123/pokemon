# Pokédex

Pokédex interactiva: lista, detalle, comparador y **combate animado estilo arcade (FIGHT!)**.

Demo: https://gonzalo123.github.io/pokemon/

## Features

- **Lista** paginada con búsqueda por nombre y filtro por tipo
- **Detalle** con sprite animado, modo shiny, grito del Pokémon y radar de stats
- **Comparador** con radar de stats superpuesto y tabla con ganadores por stat
- **Combate FIGHT!** — intro estilo Street Fighter, barras de HP con damage trail, embestidas, screen-shake, chispas, slash, daño flotante, efectividad de tipos (x2 / x0.5 / inmune) y pantalla de K.O.

## Stack

React 19 + TypeScript + Vite + framer-motion + recharts. Datos: [PokeAPI](https://pokeapi.co) (directamente desde el cliente, sin backend). Desplegada en GitHub Pages vía GitHub Actions.

## Local

```bash
cd frontend
npm install
npm run dev
```

## Human

Déjame escribir algo humano en este proyecto, quizás lo único que un humano ha escrito en este proyecto. Este proyecto es un ejercicio que hicimos en el Katayuno del 20 de junio de 2024. La idea era crear una aplicación web que mostrara información de Pokémon de manera interactiva.

En un Katayuno hacemos katas de programación un sábado por la mañana. La IA lo ha cambiado todo. Lo normal es que nunca termináramos el ejercicio, ya que lo importante del Katayuno es el debate, la conversación y la retrospectiva. Pero esta vez, por primera vez, prácticamente todos los equipos terminaron el ejercicio.

Incluso yo, que nunca he sido de frontend, he hecho esto mientras mi equipo trabajaba en otra aproximación sobre la misma idea. Vale que luego, en casa, he completado algunas cosas (lo de la lucha poco más), pero la base es lo que hicimos en el Katayuno, en un Katayuno de cuatro horas en el que la mayoría del tiempo lo hemos pasado debatiendo y discutiendo sobre cómo hacer las cosas.

El mundo de la programación ha cambiado mucho en muy poco tiempo. Quizás demasiado, pero es el mundo que nos ha tocado vivir.
